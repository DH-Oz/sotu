## 1. What this repo is

A Python package that ships every U.S. presidential **Annual Message to Congress** (1790–1946) and **State of the Union Address** (1947–present) as plain UTF-8 text inside the wheel, with a pandas-DataFrame loader and an NLTK-style file-id surface. Built once per year by scraping the UCSB Presidency Project, with the raw HTML and parsed text both committed for full provenance.

It exists so that the 2026 DH-Oz Winter School (Python/Jupyter pivot of the 2025 R/Rmd masterclass) can replace the R `sotu` CRAN package with a one-line `uv add sotu`.

## 2. Consumer contract (locked)

These columns and behaviours are fixed; downstream notebooks in `DH-Oz/CorpusAnalysis` will be written against them. **Do not regress this contract without coordinating with the masterclass repo.**

### 2.1 Must-have

A loader that returns a `pandas.DataFrame` with exactly these columns:

| Column      | Type   | Notes                                                              |
|-------------|--------|--------------------------------------------------------------------|
| `year`      | int    | SOTU year, *not* inauguration year (Washington appears in 1790).   |
| `president` | str    | **Last name only** ("Washington", "Trump"). Matches Mark's lectures. |
| `party`     | str    | E.g. "Federalist", "Democratic-Republican", "Whig", "Republican", "Democratic". |
| `sotu_type` | str    | Exactly `"spoken"` or `"written"`.                                 |
| `text`      | str    | Plain UTF-8, no extra cleaning needed by the consumer.             |

### 2.2 Nice-to-have (NLTK-style, mirrors `nltk.corpus.inaugural`)

- `fileids()` → list of `"{year}-{LastName}-{within_year_idx}"`, e.g. `"1790-Washington-1"`.
- `raw(fileid)` → str, the text of that address.
- `metadata()` → `pandas.DataFrame`, same content as the loader but exposed as an explicit accessor; `metadata().to_dict()` ↔ `fileids()` is the same data, two views.

### 2.3 Helpful but non-blocking

- `__doc__` on the package documents coverage (e.g. `"1790 through 2025"`) — generated from the data, not hand-edited.
- Package-level constant `COVERAGE = (1790, 2025)` exposed for programmatic use.

### 2.4 Public API surface

```python
import sotu

df = sotu.load()                       # pandas.DataFrame, must-have shape
df = sotu.load(full=True)              # all metadata columns (date, source_url, etc.)
ids = sotu.fileids()                   # ['1790-Washington-1', ...]
text = sotu.raw('1790-Washington-1')   # str
meta = sotu.metadata()                 # pandas.DataFrame, full schema
print(sotu.COVERAGE)                   # (1790, 2025)
```

No other public symbols. Everything else lives under `_`-prefixed modules.

## 3. Scope decisions

### 3.1 Included

- **Annual Messages to Congress (1790–1946)** — Article II § 3 ("from time to time") fulfilment. Originally delivered to a joint session by Washington and Adams; written-only from Jefferson (1801) through Taft (1912); spoken again from Wilson (1913) onward.
- **State of the Union Addresses (1947–present)** — same constitutional duty, renamed under Truman.

### 3.2 Excluded

- **Inaugural addresses** (separate corpus, already covered by `nltk.corpus.inaugural`).
- **Other joint-session addresses** that are not Annual Messages / SOTU — e.g. post-9/11 address (Bush, 2001-09-20), first-year joint-session addresses by incoming presidents (Reagan 1981, Clinton 1993, Bush 2001, Obama 2009, Trump 2017, Biden 2021, Trump 2025). These are constitutionally distinct; the R `sotu` package excludes them and we follow that precedent.
- **Farewell addresses, eulogies, fireside chats, press conferences** — out of scope.

### 3.3 Year range

- **1790 — current year**, inclusive of every Annual Message / SOTU UCSB publishes.
- Some years have **two** addresses (presidential transition years where the outgoing president delivered a final SOTU in January and the incoming president gave a separate address). This is what the `within_year_idx` in the fileid encodes.
- Some years have **zero** addresses (rare, but e.g. year of a presidential death without a successor address). The build does not error on zero-address years; the validator simply records them.

### 3.4 Spoken vs written classification

Sources of truth, in priority order:

1. **Per-address override** in `data/sotu_type_overrides.csv` (hand-curated, committed).
2. **Year-based heuristic** baked into the parser:
   - 1790–1800 → spoken
   - 1801–1912 → written
   - 1913–present → spoken (with overrides for years where a separate written version was submitted, e.g. some Nixon years).
3. If UCSB document metadata (when present) disagrees with the heuristic, the heuristic wins unless overridden in (1). The validator logs disagreements for human review.

This is deliberately a small, inspectable table rather than a complex inference rule. ~5–10 override rows expected over 235 years.

## 4. Source: UCSB Presidency Project

**Authoritative source:** <https://www.presidency.ucsb.edu/documents/presidential-documents-archive-guidebook/annual-messages-congress-the-state-the-union>

The Presidency Project (Woolley & Peters, UC Santa Barbara) is the canonical scholarly archive. It indexes every Annual Message and SOTU with structured metadata (president, date, document type) and the full transcribed text.

### 4.1 Why UCSB, not stressosaurus or the R `sotu` package as a base

The instructor explicitly chose "scrape, but reliably." Considered alternatives:

- **R `sotu` CRAN package (Arnold)** — already cleaned, MIT-licensed, parity-able. Rejected: stale (last major refresh circa 2020), upstream maintenance cadence opaque, and "scrape" was the brief.
- **`stressosaurus/raw-data-state-of-the-union`** — already has plain `.txt` extracts. Rejected: uses bs4 (instructor: "gross"), provenance chain has an extra hop, and we'd inherit whatever extraction decisions stressosaurus made.

UCSB gives us a single source of truth, current through the latest address, with reproducible scrape semantics.

### 4.2 Scraping ethics and reliability

- **User-Agent**: `sotu-corpus-builder/<version> (+https://github.com/DH-Oz/sotu)` — identifies us; gives UCSB admins a contact path.
- **Rate limit**: 1 request per 2 seconds (`asyncio.sleep` or `time.sleep` between fetches). One full build = ~250 requests = ~10 minutes wall time. Acceptable; runs once per year.
- **Retries**: 3 attempts with exponential backoff (1s, 4s, 9s) on 5xx and connection errors. Hard fail on 4xx (we have the wrong URL).
- **Idempotence**: every fetched HTML is written to `raw/ucsb/<slug>.html` before parsing. If the file exists, skip the fetch. `--refresh` flag forces re-fetch of a specific year.
- **Snapshot provenance**: `raw/` is **committed**. ~250 HTML files, ~20–50 MB. This buys:
  - Deterministic rebuilds without internet.
  - Parser bug fixes without re-scraping.
  - Auditable record of what UCSB served on a given date.
- **Determinism check**: build emits a `data/manifest.json` with `sha256(raw_html)` and `sha256(parsed_text)` per fileid. Re-running the build on the same `raw/` must produce identical `data/`. CI enforces this.

## 5. Tech stack

### 5.1 Runtime dependencies (ship with the wheel)

- **`pandas`** (>= 2.0) — required by the consumer contract. Hard dep, not optional. Students all have it via the masterclass setup anyway.
- **Python 3.10+** — `str.removeprefix`/`removesuffix`, structural pattern matching available if useful. Matches the 2026 masterclass's Python 3.14 baseline with room for users on older Pythons.

That's it for runtime. The package is a thin loader over bundled data files.

### 5.2 Build-time dependencies (dev only, not in the wheel)

- **`httpx`** — HTTP client. `httpx.Client(timeout=30, follow_redirects=True)`.
- **`selectolax`** — HTML parser. Lexbor-backed, fast, simple CSS-selector API. Replaces bs4 per instructor preference. Pure-Python wheel on all platforms.
- **`tenacity`** — retry decorator for the fetcher. Small (~30 KB), well-maintained.
- **`pytest`**, **`pytest-cov`** — tests.
- **`ruff`**, **`ty`** (or `mypy`) — lint + types.
- **`uv`** — env management; locks via `uv.lock`.

`pyproject.toml` `[project.optional-dependencies] build = [...]` exposes these so contributors can `uv sync --extra build`; end users never pull them.

### 5.3 Build backend

`hatchling`. Data files shipped via:

```toml
[tool.hatch.build.targets.wheel]
packages = ["src/sotu"]

[tool.hatch.build.targets.wheel.shared-data]
"src/sotu/data" = "sotu/data"
```

Wheel size estimate: ~5–10 MB compressed (250 speeches × ~10–30 KB each text, plus CSVs).

## 7. Data shapes

### 7.1 `src/sotu/data/metadata.csv`

```csv
fileid,year,date,president,party,sotu_type,source_url,raw_html_path,word_count,sha256
1790-Washington-1,1790,1790-01-08,Washington,Federalist,spoken,https://www.presidency.ucsb.edu/documents/...,raw/ucsb/1790-01-08-...html,1089,abc123...
1790-Washington-2,1790,1790-12-08,Washington,Federalist,spoken,...
...
```

- `fileid` — primary key, format `{year}-{LastName}-{within_year_idx}`.
- `year` — int.
- `date` — ISO 8601 (YYYY-MM-DD).
- `president` — last name only.
- `party` — joined from `presidents.csv`.
- `sotu_type` — `"spoken"` or `"written"`.
- `source_url` — UCSB document URL.
- `raw_html_path` — path within repo to the cached HTML.
- `word_count` — int, computed by parser.
- `sha256` — hex digest of the parsed text file.

### 7.2 `src/sotu/data/presidents.csv`

```csv
last_name,full_name,party,terms
Washington,George Washington,Federalist,1789-1797
Adams,John Adams,Federalist,1797-1801
Jefferson,Thomas Jefferson,Democratic-Republican,1801-1809
...
```

- One row per **last name**. If two presidents share a last name (Adams, Bush, Roosevelt, Harrison, Johnson), disambiguate via a `disambiguator` column carrying e.g. `"John Adams"` vs `"John Quincy Adams"`; the parser uses date ranges to pick the right one.

Actually — we need disambiguation built in. Final schema:

```csv
president_id,last_name,full_name,party,term_start,term_end
washington,Washington,George Washington,Federalist,1789-04-30,1797-03-04
adams-1,Adams,John Adams,Federalist,1797-03-04,1801-03-04
jefferson,Jefferson,Thomas Jefferson,Democratic-Republican,1801-03-04,1809-03-04
...
adams-6,Adams,John Quincy Adams,Democratic-Republican,1825-03-04,1829-03-04
...
roosevelt-26,Roosevelt,Theodore Roosevelt,Republican,1901-09-14,1909-03-04
roosevelt-32,Roosevelt,Franklin D. Roosevelt,Democratic,1933-03-04,1945-04-12
...
```

`president_id` is internal (e.g. `roosevelt-26` = 26th president). The public-facing `president` column on the DataFrame stays last-name-only per the consumer contract. The fileid stays last-name-only. Disambiguation is handled at *build* time by joining date ↔ term ranges; consumers never see `president_id`.

When two same-last-name presidents both appear in the corpus, the fileid pattern still works: `1825-Adams-1` is unambiguously John Quincy Adams because of the year.

### 7.3 `data/sotu_type_overrides.csv`

```csv
fileid,sotu_type,reason
1973-Nixon-1,written,UCSB document marked as written submission
...
```

Small, commented, intended to be reviewed by a human on every yearly bump.

### 7.4 `src/sotu/data/speeches/<fileid>.txt`

Plain UTF-8. Paragraph breaks preserved as `\n\n`. No HTML, no inline citations, no `[Applause]` markers (UCSB doesn't include those for historical texts anyway). Line endings normalised to `\n`. Final newline at EOF.

### 7.5 `src/sotu/data/manifest.json`

```json
{
  "build_date": "2026-05-20T14:33:21Z",
  "package_version": "0.1.0",
  "coverage": [1790, 2025],
  "total_addresses": 237,
  "files": {
    "1790-Washington-1": {
      "sha256_text": "...",
      "sha256_raw_html": "...",
      "word_count": 1089
    },
    ...
  }
}
```

Loaded at package init; powers `sotu.COVERAGE` and `sotu.__doc__`.

## 8. Build pipeline

One command for a full rebuild:

```bash
uv run sotu-build
```

(Entry point declared in `pyproject.toml` `[project.scripts]`.)

### 8.1 Stages

1. **Discover** (`tools/discover.py`)
   - Fetch the UCSB archive index page (or several index pages — UCSB paginates).
   - Parse with selectolax: extract every document link matching the Annual Message / SOTU document-type filter.
   - Cross-reference: every president × year should have ≥1 link unless we've recorded a known gap.
   - Output: `tools/_discovered_urls.json` (gitignored; intermediate artefact).

2. **Fetch** (`tools/fetch.py`)
   - For each URL in `_discovered_urls.json`:
     - Compute `raw/ucsb/<slug>.html` (slug from URL path).
     - If file exists and `--refresh` was not passed, skip.
     - Otherwise httpx GET, retry with tenacity, write atomically.
   - Rate-limit: 1 request per 2 seconds.
   - Output: populated `raw/ucsb/`.

3. **Parse** (`tools/parse.py`)
   - For each `raw/ucsb/*.html`:
     - selectolax-parse, extract title, date, body text.
     - Compute fileid from year, last name, within-year index.
     - Write `src/sotu/data/speeches/<fileid>.txt`.
     - Emit a metadata row.
   - Output: `src/sotu/data/speeches/*.txt` + intermediate metadata rows.

4. **Classify** (`tools/classify.py`)
   - Apply year-based spoken/written heuristic.
   - Apply per-address overrides from `data/sotu_type_overrides.csv`.
   - Join party from `data/presidents.csv` via term-date lookup.
   - Output: complete `src/sotu/data/metadata.csv`.

5. **Validate** (`tools/validate.py`)
   - Schema: every CSV column has the expected dtype.
   - Coverage: count addresses per decade vs expected baseline.
   - Round-trip: every fileid has both a metadata row and a `.txt` file.
   - Determinism: re-running parse → classify on the same `raw/` must produce byte-identical `data/`.
   - Manifest: regenerate `src/sotu/data/manifest.json`.
   - Output: `BUILD-OK` or a list of failures.

### 8.2 Yearly bump runbook (post-launch)

```bash
git checkout -b add-2027
uv run sotu-build --year 2027          # fetches only 2027 docs, parses, regenerates manifest
uv run pytest                          # all tests still pass
git diff src/sotu/data/                # human review of new texts
# Edit data/sotu_type_overrides.csv if classification got 2027 wrong
git add raw/ucsb/2027-*.html src/sotu/data/ data/sotu_type_overrides.csv
git commit -m "Add 2027 SOTU"
# Bump version in pyproject.toml: 0.1.0 → 0.2.0 (data change = minor)
git tag v0.2.0
git push origin add-2027 v0.2.0
gh pr create --fill
# Once merged, the release workflow publishes to PyPI on tag push.
```

Documented in `docs/CONTRIBUTING.md`.

## 9. Definition of Done (audit checklist)

Each item is observable and falsifiable.

1. `pip install sotu` from PyPI succeeds in a clean venv on Python 3.10, 3.11, 3.12, 3.13, 3.14.

## 11. Open questions for auditors

These are the things I made a judgement call on; flag any you disagree with.

1. **Commit `raw/` HTML?** I chose **yes** (provenance + deterministic rebuilds outweigh ~30 MB repo cost). Audit could argue: gitignore `raw/`, push snapshots to a separate `DH-Oz/sotu-archive` repo as releases instead. Trade-off is provenance vs main-repo bloat.

2. **Pandas as a hard runtime dep.** I chose **yes** (consumer contract demands DataFrame; masterclass students all have pandas). Audit could argue: ship a `Corpus` object with `.to_pandas()` method gated on optional `[pandas]` extra, keeping the wheel installable without pandas. Trade-off is install-size vs API ergonomics.

3. **Disambiguator strategy for shared last names.** I chose **internal `president_id` + term-date join at build time**, with the public DataFrame staying last-name-only. Audit could argue: expose a second column `president_full` for cases where users need to disambiguate Adams-from-Adams in the consumer-facing data. The consumer spec doesn't ask for it, but it's cheap to add.

4. **Spoken/written for transition periods.** The 1913 boundary (Wilson reviving spoken delivery) and 1947 (Truman SOTU rename) are inflection points. I rely on UCSB's document-type metadata + overrides. Audit could push for a primary-source citation for each classification.

5. **Annual Messages vs other joint-session addresses for first-year incoming presidents.** I exclude them per R `sotu` precedent. Some scholars would include them. Audit decision: lock the inclusion rule explicitly.

6. **Build cadence.** Manual yearly bump per instructor. Audit could push for an annual scheduled GitHub Action that opens a PR with the new year's data; instructor reviews and merges. Cheaper than a cron, still human-gated.

7. **Versioning scheme.** I propose `0.1.0` for initial release, then `0.YEAR.0` for yearly bumps once stable (e.g. `0.2026.0`). Audit could prefer SemVer-strict (each yearly bump = minor) or CalVer-strict (`2026.0.0`). The masterclass repo uses calendar-major (`v2026.x`); package versioning is independent.

8. **Coverage of "Joint Session Address" edge case for 2025.** Trump's March 2025 address was labelled by UCSB as a "joint session address" not a SOTU (because he was newly inaugurated). Do we include it? **Decision: no**, per the exclusion rule. The next true SOTU will be 2026. Audit decision: confirm by sampling UCSB document types for transition years.

## 12. Risks and mitigations

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| UCSB changes HTML structure between yearly bumps | Medium | Parser breaks | Committed `raw/` HTML lets us rebuild any year without re-fetching; parser is isolated module, fixture tests catch structural changes. |
| UCSB rate-limits or blocks our scraper | Low | One-time fetch fails | 2-second delay + UA identifying us; if blocked, we fall back to manual cookie-bearing fetch documented in CONTRIBUTING. |
| New address type emerges (e.g. a future president skips formal SOTU) | Low | Coverage validator complains | Validator warns, doesn't fail; instructor reviews on yearly bump. |
| PyPI `sotu` name claim-jumped before our first publish | Very low | Need to rename | Squat-claim `sotu` on PyPI under DH-Oz account immediately after Phase 1. |
| Pandas major version breaks our DataFrame schema | Low | Consumer notebooks break | Pin `pandas >= 2.0, < 3.0`; add upper bound on next major. |
| Disambiguation table goes wrong for an obscure year | Low | One address mis-attributed | `tests/test_classify.py` covers every known same-last-name pair; validator round-trip catches text/metadata desync. |
| Instructor forgets to bump version after data change | Medium | Stale `pip install sotu` | Pre-commit hook (or CI lint) refuses commits that change `src/sotu/data/` without bumping `pyproject.toml` version. |

## 13. Out of scope (explicit non-goals)

- **NLP preprocessing** — no tokenisation, lemmatisation, POS, stopwording, dependency parsing. That's the masterclass's job.
- **Other corpora** — inaugural addresses, fireside chats, debates. Separate packages if ever needed.
- **Non-English translations** — corpus is English-original.
- **Audio / video** — text only.
- **Sub-speech structure** — no paragraph-level metadata, no speaker turns, no annotations. Just `text`.
- **API server / web frontend** — it's a Python package, not a service.
